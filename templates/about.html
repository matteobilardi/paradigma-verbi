What is {{ app_name }} about?

It is a website that enables the user to scan a Latin text for verbs and find their paradigms. It is also my first real world coding project and my final project for CS50x.

Why did I build it?

While initially at loss for ideas of what could my CS50x final project be, the idea of a paradigm finder that analyzed a full text instead of single words struck me when I was in the process of learning the paradigms of a piece of Latin poetry. This is something that is often asked of you in Italian high schools, especially during the first few years of studying it, and it can be a rather long task: you need to identify the verb, find its base form, search for it in a dictionary, and you will have found its paradigm, repeat for every verb. Even if you are using an online dictionary, this process involves multiple steps which can be automated. From this concept {{ app_name }} was born.

Should you use it?

It might seem odd that I am posing this question myself: being the developer one would think I should obviously answer OF COURSE! It is not that simple though, for 2 main reasons:

  1. {{ app_name }} may be detrimental for you learning process as finding paradigms is something that you should really be able to do if you intend to study Latin. Using this website before you master that first is not recommended.
  2. The website is far from perfect. Not only is it my first real world coding project but, having significantly less guidance, it is much more complex than typical CS50 problem sets. False positives, for example, are often present when trying to infer whether or not a word is a verb or paradigms might sometimes be wrong. If you want to help me make it better though, I encourage you to do so via GitHub

Still, if those are non-issues to you, you definitely should use it.
